第 10 课：MBarrier 基础

对应代码：10_mbarrier.cu 需要 GPU（SM90 / Hopper）。

核心概念

MBarrier（Memory Barrier）是 Hopper 架构引入的硬件同步原语，存储在 shared memory 中的 64-bit 数据结构。与 __syncthreads 相比，它支持：

  • 部分线程同步：只让指定线程参与，不是整个 CTA
  • Transaction Bytes：与 TMA 异步搬运配合，等待指定字节数的搬运完成
  • Phase 翻转：0↔1 循环，不需要重新初始化，天然支持流水线

1. MBarrier 的 64-bit 数据结构

字段 含义
phase 当前阶段（0 或 1），所有条件满足后自动翻转
arrive_count 还差多少线程到达（初始化时设为 thread_count，每次 arrive 减 1）
transaction_count 还差多少字节完成（TMA 搬运时使用）

翻转条件：arrive_count == 0 && transaction_count == 0 → phase 自动翻转。

5. 实验 3：Phase 翻转

5.1 核心发现

Phase 在每轮 arrive 完成后自动翻转，不需要重新初始化。

本实验用 PTX mbarrier.test_wait.parity 指令直接从硬件探测真实 phase，而不是手动推算。

5.2 探测 Phase 的方法

mbarrier.test_wait.parity 的语义：检测 barrier 的 phase 是否 != 传入的 parity。

  • test_wait.parity(mbar, 0) 返回 true → phase != 0 → phase=1
  • test_wait.parity(mbar, 0) 返回 false → phase == 0

注意：mbarrier 没有"读取 phase"的直接 API，test_wait.parity 是唯一可用的探测手段。

6. MBarrier vs __syncthreads 对比

特性 __syncthreads mbarrier
同步范围 整个 CTA (block) 可选线程子集
异步搬运支持 不支持 支持 (transaction bytes)
Phase 翻转 不支持 支持 (0↔1 自动循环)
流水线支持 不支持 支持 (多 stage × 多 round)
TMA 配合 不支持 支持 (硬件自动 complete_tx)
存储位置 无需分配 shared memory (64-bit)
初始化 不需要 需要 (thread_count)
硬件支持 所有架构 SM90+ (Hopper)

7. API 总结

API 作用 使用场景
initialize_barrier(mbar, N) 创建 barrier，期望 N 个 arrive 只需调用一次
set_barrier_transaction_bytes(mbar, bytes) arrive + 设置期望搬运字节 TMA 搬运前调用
arrive_barrier(mbar) 通知 “我已到达” 线程完成工作后
wait_barrier(mbar, phase) 等待 phase 翻转 等待同步点
